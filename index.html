How I Work

Process

Every project is different. The approach isn't.

01
Discover
Stakeholder interviews, workflow observation, and competitive analysis. I spend more time understanding the problem than solving it.
02
Define
Jobs to be done, user flows, and information architecture. I document decisions so the team can debate the right things.
03
Design
Low-fi to high-fi, often in the same week. I run design sprints when speed matters and iterate slowly when it doesn't.
04
Validate
Usability tests, A/B tests, and post-launch instrumentation. I track outcomes, not just deliverables.
